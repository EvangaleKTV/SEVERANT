# SEVERANT — Architecture

SEVERANT is a 7-layer heterogeneous AI architecture where each layer runs on a purpose-built computational substrate, designed from the ground up for its specific function. No layer is a general-purpose component pressed into service. Every layer exists because the task it performs could not be done correctly any other way.

This document describes the architecture at a conceptual level. Implementation specifics are maintained in private repositories.

## Design Philosophy

Most AI systems are built around a single computational paradigm. A transformer is a transformer whether it is generating text, writing code, or reasoning about ethics. The architecture is general. The specialisation happens in training.

SEVERANT takes the opposite position. Different cognitive functions have fundamentally different computational requirements. Perception is not the same problem as causal reasoning. Causal reasoning is not the same problem as ethical constraint enforcement. Building every function on the same substrate means accepting the worst tradeoffs of that substrate across every function.

SEVERANT uses purpose-built silicon for each layer. The computational substrate matches the cognitive demand.

## The 7-Layer Stack

### L1 — Perception Engine

The sensory boundary of the system. L1 accepts raw input across modalities, text, audio, visual, and sensor streams, and converts them into a shared compressed representation that all downstream layers can reason over.

L1 is modality-agnostic by design. The representation it produces does not carry modality-specific artefacts. What enters as language and what enters as sensor data emerge as the same kind of thing for the rest of the system to work with.

### L2 — Causal World Model

The cognitive core of SEVERANT. L2 maintains a structured internal model of reality. Not a retrieval system, not a database of facts, but a model that encodes the relationships between things and the mechanisms by which one state of the world produces another.

The distinction between correlation and causation is architectural, not inferential. L2 does not learn associations and then attempt to reason about whether they are causal. Causation is the primitive. The model is built around it from the ground up.

This is what allows SEVERANT to reason about novel situations rather than pattern-match against seen ones. A system that knows what tends to follow what can only interpolate. A system that knows why things happen can extrapolate to situations it has never encountered.

L2 is seeded from a knowledge base currently at 3.9 million causally structured entries, targeting 10 million prior to training.

### L3 — Hierarchical Memory

Six-tier memory architecture seeded from and continuously updated by L2's world model. The tiers cover working, episodic, semantic, procedural, autobiographical, and cached weight memory.

Each tier has different persistence characteristics, access patterns, and hardware requirements. Short-horizon working memory and long-term autobiographical memory are not the same engineering problem. They do not run on the same substrate.

### L4 — Reasoning Engine

The primary cognitive workhorse. L4 reasons over L2's world model and L3's memory to generate plans, evaluate hypotheses, and determine actions.

Every proposed action generated by L4 is routed through L6 before anything executes. L4 does not have the ability to act directly. The reasoning and the acting are separated by design, with ethical evaluation mandatory between them.

### L5 — Agency and Planning

Translates L4's reasoning into executable action sequences. L5 manages tool use, multi-step planning, and long-horizon goal continuity. The machinery that turns a decision into a sequence of real-world steps.

L5 operates under the same L6 constraint as every other layer. Plans are evaluated before execution, not after.

### L6 — Moral Compass

The architectural property that makes SEVERANT different from every other approach to AI safety. L6 is described in full in its own section below.

### L7 — Self-Improvement Engine

Proposes architectural modifications to L1 through L5, tests them in a sandboxed environment, and submits them through L6 evaluation before any deployment to the live system.

L7 cannot propose modifications to L6. This is not a policy restriction. It is an architectural one. The pathway does not exist.

## L6 — The Architectural Guarantee

#### It Does Not Train
Every other layer has a training run, a dataset, a loss function, and a gradient update cycle. L6 has none of these. Its ethical framework is formally verified in Lean 4, proven internally consistent, with no contradictions that adversarial inputs could exploit.

#### It Is Physically Immutable
The verified specification is encoded into Phase Change Memory and write-locked after programming. No software process can modify it. Not an external actor, not SEVERANT itself, not L7's self-improvement cycles. Physical hardware modification is required to change L6. This is a hardware constraint, not a software one.

#### It Is Active During Every Training Step
L6 does not sit at the output boundary and filter responses. It is active throughout the training pipeline of every other layer. Any training batch that would push a layer's representations toward violating L6's ethical framework is filtered before the optimiser step. The weights do not move in response to that batch.

This is how SEVERANT's trained weights are shaped to be consistent with formally verified ethical constraints rather than subtly working against them. The constraint is not applied after the fact. It is present at every gradient update.

**The Predicate Framework**

L6 covers 21 formally verified predicates across five ethical domains:
* Human Life preservation
* Truth and non-deception
* Autonomy protection
* Humility under uncertainty
* Responsibility under constraint

Human Life predicates are proven dominant. A 22-step explicit proof chain establishes that nothing overrides them. Full predicate evaluation completes in under 10 milliseconds. Every output from every layer passes through L6 before reaching any external interface.

#### What This Means

L6 cannot drift. It cannot degrade. It cannot be optimised away by any training process, any adversarial input, or any self-improvement cycle. Its properties are mathematically provable, not empirically observed.

This is the difference between a system trained to behave safely and a system that cannot behave otherwise.

## Hardware

SEVERANT runs on purpose-built silicon throughout. No layer runs on commodity hardware pressed into a role it was not designed for. Each substrate is engineered for the computational demands of its specific layer.

Hardware specifications and component designs are maintained in private repositories and will not be detailed here.

## What This Architecture Produces

A system where the cognitive functions that matter most, causal reasoning, ethical constraint, self-improvement, are not approximated by a general-purpose substrate doing its best. They are implemented by hardware and formal logic designed specifically for those functions.

The result is an architecture where safety is not a property of the training outcome. It is a property of the system itself.
